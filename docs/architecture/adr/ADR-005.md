# ADR-005: Design of the movement‑cost matrix C (biomechanical model)

**Status**: Accepted

**Context**
We need a matrix C where C_ab predicts the time or effort to move from keyboard state a to state b. Early versions used simple Euclidean distance; this ignored known asymmetries such as finger‑change penalties, hand switches, modifier activation, and posture costs (wrist deviation, finger joint excursions, shoulder/torso movement). Consequently, optimisations produced layouts that felt awkward despite low geometric cost.

**Decision**
Model C as the sum of four empirically grounded components:

1. **Geometric Fitts term** (derived from Fitts’ law and its extensions):
   \[
   D_{ab}= \sqrt{w_x (\Delta x)^2 + w_y (\Delta y)^2}
          + \lambda_h \Delta_h
          + \lambda_f (1-s_{ab})
          + \lambda_m \Delta_m,
   \]
   where  
   - Δx, Δy are the horizontal and vertical key centre displacements (mm).  
   - w_x, w_y scale horizontal vs. vertical motion (obtained from separate horizontal/vertical Fitts fits).  
   - Δ_h = 1 if hand changes, else 0 (hand‑change penalty λ_h).  
   - s_ab = 1 if the same finger is used, else 0 (finger‑change penalty λ_f).  
   - Δ_m = 1 if modifier state changes, else 0 (modifier‑change penalty λ_m).

2. **Effective target width** to refine the index of difficulty:
   \[
   \frac{1}{W_e}= \frac{1}{W_0}\Bigl[1
          + \kappa_w |\theta|
          + \kappa_{wf} (1-s_{ab})
          + \kappa_{wh} \Delta_h
          + \kappa_{wm} \Delta_m\Bigr],
   \]
   with θ = atan2(Δy, Δx) (approach angle), W₀ the nominal key width (~19 mm), and κ parameters obtained from width‑adjustment experiments.

3. **Fitts index of difficulty**:
   \[
   \text{ID}_{ab}= \log_2\!\left(\frac{D_{ab}}{W_e}+1\right).
   \]

4. **Postural / joint‑effort term** (linear approximation of muscular effort):
   \[
   P_{ab}= \alpha_w |\theta_w|
          + \alpha_f \sum_{j∈{MCP,PIP,DIP}} |Δφ_j|
          + \alpha_h |Δθ_{humerus}|
          + \alpha_t |Δθ_{torso}|,
   \]
   where the angle terms are obtained via a 2‑D inverse‑kinematics model of the finger‑hand‑arm linkage given the start and end key centres and the prescribed finger/hand assignments. The α coefficients are derived from linear regression of measured movement time against joint excursions in a calibration study.

5. **Base actuation cost** c₀ (average key‑down/up time for a stationary finger), added to account for the intrinsic time to depress and release a key.

The final predicted movement time is:
\[
C_{ab}= \frac{1}{TP}\,\text{ID}_{ab} \;+\; P_{ab} \;+\; c_0,
\]
where TP (throughput in bits/s) is the slope of the regression between ID and observed movement time in the baseline Fitts task.

**Alternatives Considered**
1. **Pure Euclidean distance** – neglects finger/hand/modifier effects and posture; produced unsatisfactory layouts (see early experiments).
2. **Weighted Manhattan distance** – still isotropic; cannot capture direction‑specific costs like wrist deviation.
3. **Lookup table from extensive motion‑capture data** – would require collecting data for every possible pair of states (M²), infeasible for large M, and would not generalise to new keyboards or users.
4. **Addition of interaction terms (e.g., wrist·finger)** – increases model complexity; initial experiments showed that the additive model already achieved R² > 0.85. Adding interactions yielded diminishing returns while complicating calibration.
5. **Use of Fitts’ law with a fixed effective width** – ignored the influence of approach angle and effector changes on the perceived target size, leading to systematic bias for diagonal moves.

**Consequences**
- **Positive**: C now reflects the main biomechanical factors known to affect finger‑to‑key movement time. The model is linear in a small set of interpretable parameters, all of which can be measured in a short (~15 min) calibration session or taken from published literature.
- **Negative**: The model assumes independence of the additive terms; interactions may exist but are omitted for simplicity. If future validation shows systematic residuals, the model can be extended.
- **Implementation Impact**: Files `src/core/kb_builder.py` (geometry preprocessing and pair generation) and `src/core/kinematics.py` (inverse‑kinematics and angle computation) implement the formulae. Parameters are loaded from `src/config/calibration.json` (if present) otherwise fall back to literature defaults. The resulting matrix is stored as a NumPy dense array; for large keyboards a sparsity threshold (e.g., ignore pairs with Chebyshev distance > 2) can be applied without affecting correctness for typical fingering spans.
